# Dashboard UI Polish: Nav, Sidebar, iPad & Modal Fix

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix 4 dashboard UI issues — restore the transparent order modal background, remove the duplicate mobile hamburger, restructure the sidebar into an OWA-style hierarchy with iPad icon-only mode, and mirror that exact hierarchy in the account dropdown.

**Architecture:** All changes are confined to 5 existing files; no new files are created. **Tech Stack:** Next.js App Router · Tailwind 4 CSS variable syntax (`bg-(--var)`) · shadcn/ui · Lucide icons

### Task 1: Fix Order Detail Modal Transparent Background

**Files:**
- Modify: `src/app/dashboard/components/order-detail/Dashboard-OrderDetailDialog.tsx:139`

**Context:** Tailwind 4 resolves `bg-{name}` to `background-color: var(--color-{name})`. `globals.css` defines `--popover` and `--popover-foreground` (shadcn convention) but NOT `--color-popover` or `--color-popover-foreground`. So `bg-popover` and `text-popover-foreground` have no value and the modal is transparent.

- [ ] **Step 1: Fix the class string on `DialogPrimitive.Popup` (line 139)**

```tsx

```

Replace with:
```tsx
<DialogPrimitive.Popup className="fixed top-1/2 left-1/2 z-50 w-[calc(100%-2rem)] sm:max-w-4xl max-h-[90dvh] -translate-x-1/2 -translate-y-1/2 rounded-xl bg-(--color-background) text-sm text-(--color-foreground) ring-1 ring-(--color-border) outline-none flex flex-col duration-100 data-open:animate-in data-open:fade-in-0 data-open:zoom-in-95 data-closed:animate-out data-closed:fade-out-0 data-closed:zoom-out-95">
```

The three changes: `bg-popover` → `bg-(--color-background)`, `text-popover-foreground` → `text-(--color-foreground)`, `ring-foreground/10` → `ring-(--color-border)`.

### Task 4: Restructure Sidebar Hierarchy + iPad Icon-Only Mode

**Files:**
- Modify: `src/app/dashboard/components/Dashboard-Sidebar.tsx`

**Context:** Three changes in one file:
1. Remove the section header text labels ("Orders", "Insights", "Admin").
2. Orders becomes a clickable parent link (→ `?view=orders`) with Calendar/Complete/Archive as visually distinct sub-links — text-only, indented, muted — like OWA.
3. Breakpoint behavior: `md` (768px–1023px) = icon-only narrow sidebar (`w-14`); `lg` (1024px+) = full-width sidebar with labels (`w-56`). Sub-links at `md` show their own icons; at `lg` they're text-only and indented (no icon), matching OWA exactly.

The `title={label}` attribute on each link provides a browser tooltip at icon-only width for accessibility.

- [ ] **Step 1: Full replacement of Dashboard-Sidebar.tsx**

Replace the entire file content with:

```tsx
"use client"

import Link from "next/link"
import { useSearchParams } from "next/navigation"
import { cn } from "@/lib/utils"
import { LayoutGrid, Calendar, CheckCircle, Archive, BarChart2, Users, Settings } from "lucide-react"

type Props = { role: string }

export default function DashboardSidebar({ role }: Props) {
  const searchParams = useSearchParams()
  const view = searchParams.get("view") ?? "orders"
  const isAdmin = role === "admin"

  return (
    <aside className="hidden md:flex flex-col w-14 lg:w-56 shrink-0 border-r border-(--color-border) bg-(--color-background) h-[calc(100dvh-3.5rem)] sticky top-14 overflow-y-auto p-2 gap-1">
      <SidebarLink href="?view=orders" label="Orders" icon={<LayoutGrid size={16} />} active={view === "orders"} />
      <SubLink href="?view=calendar" label="Calendar" icon={<Calendar size={14} />} active={view === "calendar"} />
      <SubLink href="?view=complete" label="Complete" icon={<CheckCircle size={14} />} active={view === "complete"} />
      <SubLink href="?view=archive" label="Archive" icon={<Archive size={14} />} active={view === "archive"} />

      {isAdmin ? (
        <>
          <div className="my-1 mx-1 border-t border-(--color-border)" />
          <SidebarLink href="?view=insights" label="Insights" icon={<BarChart2 size={16} />} active={view === "insights"} />
          <SidebarLink href="?view=users" label="Users" icon={<Users size={16} />} active={view === "users"} />
          <SidebarLink href="?view=settings" label="Settings" icon={<Settings size={16} />} active={view === "settings"} />
        </>
      ) : null}
    </aside>
  )
}

type LinkProps = { href: string; label: string; icon: React.ReactNode; active: boolean }

function SidebarLink({ href, label, icon, active }: LinkProps) {
  return (
    <Link
      href={href}
      title={label}
      className={cn(
        "flex items-center gap-3 px-3 py-2 rounded-md text-sm font-medium transition-colors motion-reduce:transition-none",
        "justify-center lg:justify-start",
        active
          ? "bg-(--color-primary) text-(--color-primary-foreground)"
          : "text-(--color-foreground) hover:bg-(--color-surface)"
      )}
    >
      {icon}
      <span className="hidden lg:inline">{label}</span>
    </Link>
  )
}

type SubLinkProps = { href: string; label: string; icon: React.ReactNode; active: boolean }

function SubLink({ href, label, icon, active }: SubLinkProps) {
  return (
    <Link
      href={href}
      title={label}
      className={cn(
        "flex items-center rounded-md text-sm transition-colors motion-reduce:transition-none",
        // md: centered icon only
        "justify-center px-3 py-1.5",
        // lg: indented text only (no icon shown)
        "lg:justify-start lg:pl-8 lg:pr-3",
        active
          ? "text-(--color-primary) font-semibold"
          : "text-(--color-muted) hover:text-(--color-foreground) hover:bg-(--color-surface)"
      )}
    >
      {/* Icon visible at md (icon-only mode), hidden at lg */}
      <span className="lg:hidden">{icon}</span>
      {/* Label visible at lg (text-only), hidden at md */}
      <span className="hidden lg:inline">{label}</span>
    </Link>
  )
}
```
